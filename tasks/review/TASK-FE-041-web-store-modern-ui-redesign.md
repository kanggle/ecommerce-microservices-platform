# Goal

web-store의 현재 인라인 스타일 기반의 기본적인 UI를 모던 이커머스 사이트 수준의 세련된 디자인으로 전면 개선한다.

현재 문제:
1. CSS 프레임워크 없이 인라인 스타일만 사용하여 일관성 없는 디자인
2. 글로벌 스타일(타이포그래피, 컬러, 스페이싱) 미정의
3. 헤더/네비게이션/푸터 레이아웃 부재
4. 반응형 디자인 미적용
5. 호버/트랜지션 등 인터랙션 피드백 없음
6. 전체적으로 완성도가 낮은 프로토타입 수준

# Scope

## 포함
- CSS Module 기반 글로벌 스타일 시스템 도입 (globals.css)
- 공통 레이아웃 (Header + Navigation + Footer) 구현
- 모든 페이지/컴포넌트 UI 개선:
  - 홈페이지 (히어로 배너 + 인기 상품 그리드)
  - 상품 목록/검색 결과 (카드 그리드 + 필터 사이드바)
  - 상품 상세 (이미지 갤러리 + 정보 패널)
  - 장바구니 (아이템 리스트 + 요약 사이드바)
  - 로그인/회원가입 (센터 정렬 폼)
  - 주문 내역 (카드 기반 목록)
  - 마이페이지 (프로필, 주소)
- 반응형 디자인 (모바일/태블릿/데스크탑)
- 호버, 포커스, 트랜지션 등 마이크로 인터랙션

## 제외
- 새로운 기능 추가 (기존 기능의 UI만 개선)
- 백엔드 API 변경
- 새로운 npm 패키지 추가 (CSS-in-JS 라이브러리 등) — 기존 CSS Module + globals.css로 구현
- @repo/ui 공유 패키지 변경

# Acceptance Criteria

- [ ] globals.css에 디자인 토큰(색상, 타이포그래피, 스페이싱) 정의
- [ ] 모든 페이지에 공통 Header(로고, 네비게이션, 장바구니 아이콘, 로그인) + Footer 적용
- [ ] 홈페이지에 히어로 섹션과 인기 상품 그리드 표시
- [ ] 상품 카드에 호버 효과(그림자, 스케일) 적용
- [ ] 상품 상세 페이지 2단 레이아웃 (이미지 + 정보)
- [ ] 장바구니 페이지 정리된 테이블/리스트 + 주문 요약
- [ ] 검색바 개선 (둥근 모서리, 아이콘, 포커스 효과)
- [ ] 로그인/회원가입 폼 센터 정렬 + 카드 스타일
- [ ] 주문 내역 카드 기반 레이아웃
- [ ] 모바일 반응형 (768px 이하에서 1단 레이아웃)
- [ ] 기존 테스트 통과 유지

# Edge Cases

- 상품 이미지가 없는 경우 플레이스홀더 표시
- 매우 긴 상품명 텍스트 오버플로우 처리 (말줄임)
- 빈 장바구니, 빈 주문 내역 등 empty state 디자인
- 품절 상품의 시각적 구분

# Failure Scenarios

- CSS 로딩 실패 시에도 콘텐츠 접근 가능 (progressive enhancement)
- 이미지 로딩 실패 시 fallback 표시 (기존 ProductImage 컴포넌트 활용)
